# AnomalyX v1.0 — SOC Intelligence Dashboard

A network traffic anomaly detection tool I built using the NSL-KDD dataset and a Random Forest classifier. Started as a basic binary classifier, v1 adds the intelligence layer on top — attack categorization, MITRE mapping, behavior analysis. Built for SOC-relevant analysis, not just binary classification.

## What is does 
- MITRE ATT&CK technique mapping for each detected attack class
- Source behavior panel — failed logins, brute-force signals, root shell activations, scan-like bursts, SYN error rates
- Top targeted services and protocol breakdown for attack traffic
- Auto-generated incident summary + FP/FN explanation

## Features

### Core ML
- Random Forest classifier (100 trees, NSL-KDD)
- Binary prediction: Normal / Attack
- Accuracy, Precision, Recall, F1-Score
- Confusion matrix with TP/TN/FP/FN

## Architecture

```
app.py                         — Flask server, /api/predict returns full intel payload
network_anomaly_detector.py    — ML + attack categorization + MITRE mapping + behavior analysis
templates/index.html           — 6-tab dashboard (Results, Distribution, Alerts, Behavior, MITRE, Incident)
models/                        — Saved RF model, encoders, scaler (generated by train.py)
Dataset/                       — NSL-KDD train/test files
```

## Model Details

- **Algorithm**: Random Forest (sklearn), 100 estimators, no depth limit
- **Features**: 41 NSL-KDD features (duration, protocol, service, flag, bytes, connection stats, error rates)
- **Categorical encoding**: LabelEncoder per feature (protocol_type, service, flag)
- **Feature scaling**: StandardScaler
- **Training set**: KDDTrain+ (~125k samples)
- **Test set**: KDDTest+ (~22k samples)
- **Typical accuracy**: ~99% on KDDTest+

## Attack Categories

| Category | Description | Severity | Example Attacks |
|---|---|---|---|
| DoS | Denial of Service | HIGH | neptune, smurf, teardrop |
| Probe | Reconnaissance / Scanning | MEDIUM | nmap, portsweep, ipsweep |
| R2L | Remote-to-Local | HIGH | guess_passwd, ftp_write, imap |
| U2R | User-to-Root | CRITICAL | buffer_overflow, rootkit, ps |

## Setup

```bash
# Install dependencies
pip install flask pandas scikit-learn joblib

# Train the model (one-time)
cd Dataset
python ../train.py --train KDDTrain+.txt --test KDDTest+.txt

# Start the app
python app.py
```

Then open http://localhost:5000 and upload a .txt/.csv file contailing network logs.
